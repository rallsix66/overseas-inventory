# UI 设计规则

## 设计原则

- 面向运营人员，优先信息密度、清晰度和少点击
- 数据列表使用表格；卡片仅用于统计指标
- 只做桌面端，最小宽度 `1024px`
- 无暗色模式、复杂动画、拖拽、无限滚动和自定义滚动条
- 优先复用 shadcn/ui，禁止重复创建基础 UI 组件

## 布局与间距

- 页面外边距使用 `px-6`
- 区块间距使用 `mb-5`
- 组件间距使用 `mb-4`
- 常规元素间距使用 `gap-3`
- 紧凑元素间距使用 `gap-2`
- Dashboard 侧边栏宽度固定为 `220px`

## 表格

- 表头使用浅灰背景、小号中等字重
- 数据行保持紧凑，hover 使用浅灰背景
- 最后一列放操作按钮并右对齐
- 默认每页 20 条
- 无数据时显示说明和必要的操作引导

## 表单与操作

- 标签放在输入框上方，必填字段标记 `*`
- 输入错误显示在字段下方
- 新增和编辑表单优先使用右侧 Sheet
- 危险操作和状态变更使用确认 Dialog
- 操作按钮提交期间必须禁用并显示 loading

## 页面状态

- 数据加载使用 Skeleton，不使用全屏 loading
- 成功和失败操作使用 Toast 反馈
- 必须处理空数据、加载、错误和无权限状态
- 具体页面字段、状态颜色和交互以 `docs/page-specification.md` 及对应业务规则为准
