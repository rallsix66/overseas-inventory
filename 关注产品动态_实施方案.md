# DIS 关注产品动态 — 实施方案（4 阶段拆分版）

> 供 Codex 架构审核 + Claude 落盘执行。
> 编写：巴蒂 | 日期：2026-06-25
> 本版按 Codex 拆分意见重写：不新建 variant_follows、复用 user_variant_preference、
> 拆成 A/B/C/D 四阶段、阶段 B 告警用 safety_stock 临时规则、阶段 C 升级动态告警。

## 一、背景与需求

当前 Dashboard 仅有三个统计卡片（海外低库存 / 国内低库存 / 在途数量），运营要了解具体 SKU 状态需手动翻列表页。需要"特别关注"功能让运营登录即见重点 SKU 状态，并最终（阶段 C）基于 BigSeller 预测数据 + 各国补货周期做动态告警。

### 1.1 功能定义（最终形态）

| 维度 | 说明 |
|------|------|
| 关注对象 | `product_variant`（某仓某 SKU） |
| 关注主体 | 每个用户独立关注列表，Admin/Operator 都只能通过普通 UI 操作自己的关注 |
| 数据模型 | 复用 P5-SY11G 的 `user_variant_preference` 表，`preference_type='favorited'` |
| 仓库可见性（阶段 D） | admin 分配每个 operator 可见仓库；operator 只看被分配仓库的数据 |
| 展示形态 | Dashboard 新增"关注产品动态"区：告警条 + variant 行 |
| 告警依据（阶段 C 起） | `est_days < warehouse.lead_time_days` 为主，`product.safety_stock` 兜底 |

### 1.2 关键业务决策（全阶段保留）

- 关注粒度是 `product_variant`，不做 product 级关注
- 每个用户独立关注，Admin/Operator 都只能通过普通 UI 操作自己的关注
- 关注不影响同步、不影响库存写入、不影响他人视图
- 动态告警公式（阶段 C 实现）：`est_days < lead_time_days OR quantity < safety_stock`
- BigSeller 只取 `daily_sales` + `est_days`，不取 transit（用 shipment 模块）和 alert_stock（用 safety_stock）
- 补货周期按国家物流渠道（含缓冲）：泰国18天/马来20天/菲律宾28天/越南16天，印尼暂无数据留 null

### 1.3 架构边界（全文档约束）

1. **不新建 `variant_follows` 表**。特别关注复用 P5-SY11G 的 `user_variant_preference` 表。
2. **不依赖 `product_variant.is_archived`**。该列是遗留列，不作为业务过滤或权限条件。所有 RLS 设计不得引用 `is_archived`。
3. **关注功能与归档功能共用同一张偏好表**，通过 `preference_type` 区分：`'archived'`（归档）/ `'favorited'`（特别关注）。
4. **所有数据访问必须经过 Repository 层**。页面、组件、Server Action 不得直接调用 `createClient()` 或 `supabase.from()`。下文示例代码中的直接查询仅示意数据流和字段关系，实际实现必须封装在 Repository（如 `preferencesRepository` 或现有 `variantRepository`）中，遵循 `page → Server Action → repository → Supabase` 链路。Dashboard 查询函数（`getFollowedVariantsBasic` / `getFollowedVariantsDynamic`）在实现时需拆分为：Repository 方法（数据查询）+ Server Action 或 Server Component（调用方）。

## 二、四阶段拆分总览

| 阶段 | 名称 | 内容 | 前置依赖 |
|------|------|------|----------|
| A | P5-SY11G 收口 | 个人归档偏好：建 `user_variant_preference` 表，`preference_type IN ('archived')`；移除旧 `is_archived` RLS | 无（当前正在做） |
| **B** | **特别关注最小闭环** | 扩展 `preference_type` 加 `'favorited'`；星标按钮；Dashboard 关注区基础展示；**告警临时用 `product.safety_stock`** | 阶段 A 完成 |
| C | 动态告警 | `inventory.daily_sales` / `inventory.est_days` / `warehouse.lead_time_days`；sync RPC + Python 透传；Dashboard 告警升级为 `est_days < lead_time_days` | 阶段 B 完成 |
| D | 仓库分配权限 | `user_warehouses` 表 + operator 仓库可见性 RLS 收紧；单独评审 | 阶段 B 完成（与 C 无强依赖） |

### ⚠️ 阶段 B / 阶段 C 告警边界（重要）

**阶段 B 不实现动态告警。** 阶段 B 不新增 `inventory.daily_sales` / `inventory.est_days` / `warehouse.lead_time_days`，不改 sync RPC，不改 Python 同步透传。

因此阶段 B 如需展示告警，**只能使用已有字段 `product.safety_stock` 兜底**：
```
isLowStock = quantity < product.safety_stock
```

**这不是最终动态告警逻辑，也不是 bug，而是阶段顺序导致的临时规则。** 阶段 B 的目的是先跑通"关注 + Dashboard 展示"闭环，告警精度先妥协。

**阶段 C 完成后，Dashboard 告警升级为：**
```
isLowStock =
  (est_days IS NOT NULL AND lead_time_days IS NOT NULL AND est_days < lead_time_days)  -- 动态主告警
  OR (quantity < product.safety_stock)                                                  -- 兜底
```

阶段 C 落地后，`quantity < safety_stock` 退化为兜底，主告警切换为 `est_days < lead_time_days`。

### 4.4 `repository.ts`（关注部分 + Dashboard 查询）

> **架构约束**：以下方法封装在 Repository 内，是项目中唯一可以调用 `createClient()` 和 `supabase.from('user_variant_preference')` 的位置。页面、组件和 Server Action 不得直接访问 Supabase。

#### 4.4.1 错误类型与 Result 约定

```typescript
// types.ts — 偏好操作错误码
export type PreferenceErrorCode =
  | 'VARIANT_NOT_FOUND'      // variantId 对应的 ProductVariant 不存在
  | 'ALREADY_FAVORITED'      // 已关注（幂等，不算错误但单独标明）
  | 'NOT_FAVORITED'           // 未关注（取消关注时不存在偏好记录）
  | 'RLS_REJECTED'            // RLS 拒绝写入（用户无权操作该偏好）
  | 'DB_ERROR';               // 未知数据库错误

export class PreferenceError extends Error {
  constructor(
    public readonly code: PreferenceErrorCode,
    message: string,
  ) {
    super(message);
    this.name = 'PreferenceError';
  }
}

export type PreferenceResult<T> =
  | { success: true; data: T }
  | { success: false; error: PreferenceError };

// 中文错误映射
export function preferenceErrorMessage(code: PreferenceErrorCode): string {
  switch (code) {
    case 'VARIANT_NOT_FOUND': return '该 SKU 不存在';
    case 'ALREADY_FAVORITED': return '已关注该 SKU';
    case 'NOT_FAVORITED':     return '未关注该 SKU';
    case 'RLS_REJECTED':      return '无权操作该 SKU';
    case 'DB_ERROR':          return '数据库错误，请稍后重试';
  }
}
```

#### 4.4.2 Repository 方法

```typescript
// 正式实现时必须使用项目 Database 类型（src/types/database.ts），不得使用 any。
// 以下示例中的 Supabase join 展开类型引用自项目 Database['public']['Tables']。

/** Supabase join 展开后的 inventory 行（阶段 B 字段子集） */
type InventoryJoinRowB = {
  quantity: number;
  last_sync_at: string | null;
  warehouse: {
    id: string; name: string; country: string; type: string;
  } | null;
};

export const preferencesRepository = {
  /** 获取用户关注的 variant_id 集合 */
  async getFavoritedVariantIds(userId: string): Promise<Set<string>> {
    const supabase = await createClient();
    const { data, error } = await supabase
      .from('user_variant_preference')
      .select('variant_id')
      .eq('user_id', userId)
      .eq('preference_type', 'favorited');

    if (error) throw new PreferenceError('DB_ERROR', `查询关注列表失败: ${error.message}`);
    return new Set((data ?? []).map((r) => r.variant_id));
  },

  /** 是否已关注 */
  async isFavorited(userId: string, variantId: string): Promise<boolean> {
    const supabase = await createClient();
    const { data, error } = await supabase
      .from('user_variant_preference')
      .select('id')
      .eq('user_id', userId)
      .eq('variant_id', variantId)
      .eq('preference_type', 'favorited')
      .maybeSingle();

    if (error) throw new PreferenceError('DB_ERROR', `查询关注状态失败: ${error.message}`);
    return !!data;
  },

  /** 关注 variant */
  async favorite(userId: string, variantId: string): Promise<PreferenceResult<{ favorited: true }>> {
    // 1. 验证 Variant 存在
    const variantExists = await this._variantExists(variantId);
    if (!variantExists) {
      return { success: false, error: new PreferenceError('VARIANT_NOT_FOUND', '该 SKU 不存在') };
    }

    // 2. 幂等检查
    const already = await this.isFavorited(userId, variantId);
    if (already) {
      return { success: false, error: new PreferenceError('ALREADY_FAVORITED', '已关注该 SKU') };
    }

    // 3. INSERT
    const supabase = await createClient();
    const { error } = await supabase
      .from('user_variant_preference')
      .insert({ user_id: userId, variant_id: variantId, preference_type: 'favorited' });

    if (error) {
      // 23505 = unique_violation（已被并发插入）
      if (error.code === '23505') {
        return { success: false, error: new PreferenceError('ALREADY_FAVORITED', '已关注该 SKU') };
      }
      // 42501 = insufficient_privilege（RLS 拒绝）
      if (error.code === '42501') {
        return { success: false, error: new PreferenceError('RLS_REJECTED', '无权操作该 SKU') };
      }
      return { success: false, error: new PreferenceError('DB_ERROR', `数据库错误: ${error.message}`) };
    }

    return { success: true, data: { favorited: true } };
  },

  /** 取消关注 */
  async unfavorite(userId: string, variantId: string): Promise<PreferenceResult<{ favorited: false }>> {
    // 幂等检查
    const already = await this.isFavorited(userId, variantId);
    if (!already) {
      return { success: false, error: new PreferenceError('NOT_FAVORITED', '未关注该 SKU') };
    }

    const supabase = await createClient();
    const { error } = await supabase
      .from('user_variant_preference')
      .delete()
      .eq('user_id', userId)
      .eq('variant_id', variantId)
      .eq('preference_type', 'favorited');

    if (error) {
      if (error.code === '42501') {
        return { success: false, error: new PreferenceError('RLS_REJECTED', '无权操作该 SKU') };
      }
      return { success: false, error: new PreferenceError('DB_ERROR', `数据库错误: ${error.message}`) };
    }

    return { success: true, data: { favorited: false } };
  },

  /** 切换关注状态 */
  async toggleFavorite(userId: string, variantId: string): Promise<PreferenceResult<{ favorited: boolean }>> {
    // 1. 验证 Variant 存在
    const variantExists = await this._variantExists(variantId);
    if (!variantExists) {
      return { success: false, error: new PreferenceError('VARIANT_NOT_FOUND', '该 SKU 不存在') };
    }

    // 2. 查询当前状态
    const isFavorited = await this.isFavorited(userId, variantId);

    // 3. 执行操作并返回结果
    if (isFavorited) {
      return this.unfavorite(userId, variantId) as Promise<PreferenceResult<{ favorited: boolean }>>;
    }
    // favorite 返回 { favorited: true }，在此适配为 { favorited: boolean }
    const result = await this.favorite(userId, variantId);
    if (!result.success) return result;
    return { success: true, data: { favorited: true } };
  },

  /** 内部：校验 variantId 是否对应真实 ProductVariant */
  async _variantExists(variantId: string): Promise<boolean> {
    const supabase = await createClient();
    const { data, error } = await supabase
      .from('product_variant')
      .select('id')
      .eq('id', variantId)
      .maybeSingle();
    if (error) throw new PreferenceError('DB_ERROR', `校验 SKU 失败: ${error.message}`);
    return !!data;
  },

  /**
   * [Dashboard] 获取当前用户关注 Variant 的基础库存视图。
   * 阶段 B 临时告警：quantity < product.safety_stock（非动态告警）。
   * 阶段 C 升级为动态告警（见 getFollowedVariantsDynamic）。
   *
   * ⚠️ 查询失败时抛出 PreferenceError（不静默返回 []）。
   *    仅当用户确实没有关注任何 Variant 时才返回 []。
   */
  async getFollowedVariantsBasic(userId: string): Promise<FollowedVariantBasic[]> {
    const supabase = await createClient();
    const { data, error } = await supabase
      .from('user_variant_preference')
      .select(`
        variant:variant_id (
          id, country,
          product:product_id (id, name, code, safety_stock),
          inventory:inventory (
            quantity, last_sync_at,
            warehouse:warehouse_id (id, name, country, type)
          )
        )
      `)
      .eq('user_id', userId)
      .eq('preference_type', 'favorited');

    // 查询失败 → 抛出错误，由调用方处理（显示错误状态或降级 UI）
    if (error) {
      throw new PreferenceError('DB_ERROR', `查询关注列表失败: ${error.message}`);
    }

    // null/undefined data 视为空列表（用户无关注）
    if (!data || data.length === 0) return [];

    // 阶段 B 不做仓库权限过滤（阶段 D 才引入）
    const results: FollowedVariantBasic[] = [];
    for (const f of data) {
      const variant = unwrapJoin(f.variant);
      if (!variant) continue;
      const product = unwrapJoin(variant.product);
      if (!product) continue;
      // 显式类型：正式实现使用项目 Database 类型替代
      const inventoryRows: InventoryJoinRowB[] = (variant.inventory ?? []) as unknown as InventoryJoinRowB[];

      for (const inv of inventoryRows) {
        const wh = unwrapJoin(inv.warehouse);
        if (!wh) continue;

        const safetyStock = product.safety_stock ?? 0;
        const isLow = inv.quantity < safetyStock;  // 阶段 B 临时规则

        results.push({
          variantId: variant.id,
          productName: product.name,
          productCode: product.code,
          country: variant.country,
          warehouseId: wh.id,
          warehouseName: wh.name,
          quantity: inv.quantity,
          safetyStock,
          isLowStock: isLow,
          alertReason: isLow ? `低于安全线 ${safetyStock}` : null,
        });
      }
    }

    results.sort((a, b) => Number(a.isLowStock) < Number(b.isLowStock) ? 1 : -1);
    return results;
  },
};
```

### 4.5 `actions.ts`

```typescript
'use server';
import { revalidatePath } from 'next/cache';
import { requireActiveAuth } from '@/lib/auth';
import { preferencesRepository } from './repository';
import { toggleFavoriteSchema } from './schema';
import { preferenceErrorMessage, type PreferenceErrorCode } from './types';

export async function toggleFavoriteAction(variantId: string) {
  const user = await requireActiveAuth();

  // 1. Zod 校验
  const parsed = toggleFavoriteSchema.safeParse({ variantId });
  if (!parsed.success) return { success: false, error: '无效的 SKU ID' };

  // 2. 阶段 B 不做仓库权限校验（阶段 D 才引入 user_warehouses）
  //    operator 可关注任何 variant，Dashboard 显示其关注列表
  const result = await preferencesRepository.toggleFavorite(user.id, parsed.data.variantId);

  // 3. 操作失败 → 返回中文错误，不刷新页面
  if (!result.success) {
    return {
      success: false,
      error: preferenceErrorMessage(result.error.code),
      code: result.error.code as PreferenceErrorCode,  // 可选，供 UI 差异化处理
    };
  }

  // 4. 只有真实成功后刷新相关页面
  revalidatePath('/dashboard');
  revalidatePath('/dashboard/inventory/overseas');

  return { success: true, isFavorited: result.data.favorited };
}
```

> **架构说明**：
> - Server Action 不直接调用 `createClient()` 或 `supabase.from()`，所有数据访问通过 `preferencesRepository`。
> - `revalidatePath` 仅在确认写入成功后才执行，失败路径不触发不必要的页面刷新。
> - `variantId` 对应的 ProductVariant 存在性由 Repository 层（`_variantExists`）校验，不在 Action 层重复。
> - 错误码透传给 UI，可选用于差异化提示（如"已关注该 SKU"可显示为 toast 而非 error）。

### 4.8 Dashboard 关注区（阶段 B 版）— 调用方

**⚠️ 阶段 B 告警逻辑：临时用 `product.safety_stock`，非动态告警。**

> **架构说明**：实际查询逻辑封装在 `preferencesRepository.getFollowedVariantsBasic(userId)`（见 §4.4）。Dashboard Server Component 或 Server Action 通过 Repository 获取数据，不直接调用 `createClient()` 或 `supabase.from()`。以下为调用方类型定义和使用示例。

```typescript
// types.ts — Dashboard 关注区返回类型（阶段 B）
export interface FollowedVariantBasic {
  variantId: string;
  productName: string;
  productCode: string;
  country: string;
  warehouseId: string;
  warehouseName: string;
  quantity: number;
  safetyStock: number;        // product.safety_stock（阶段 B 唯一告警依据）
  isLowStock: boolean;        // 临时规则：quantity < safetyStock（非动态告警，不是 bug）
```

```typescript
/**
 * 动态告警判断（阶段 C 升级版，两层取严）。
 * - 主：est_days < lead_time_days（可售天数 < 补货周期）
 * - 兜底：quantity < safety_stock
 *
 * lead_time_days 为 null（发货源/未设）时，跳过主告警，仅走兜底。
 * est_days 为 null（BigSeller 无预测）时，跳过主告警，仅走兜底。
 */
export function isLowStock(
  quantity: number,
  estDays: number | null,
  leadTimeDays: number | null,
  safetyStock: number,
): boolean {
  if (estDays !== null && leadTimeDays !== null && estDays < leadTimeDays) return true;
  if (quantity < safetyStock) return true;
  return false;
}
```

**⚠️ 阶段 B 的 `isLowStock` 签名只有 `(quantity, safetyStock)`，阶段 C 升级为 `(quantity, estDays, leadTimeDays, safetyStock)`。** 调用方需同步更新。

```typescript
// preferencesRepository 新增方法（阶段 C）

/** Supabase join 展开后的 inventory 行（阶段 C 字段子集，含 daily_sales/est_days/lead_time_days） */
type InventoryJoinRowC = {
  quantity: number;
  last_sync_at: string | null;
  daily_sales: number | null;
  est_days: number | null;
  warehouse: {
    id: string; name: string; country: string; type: string;
    lead_time_days: number | null;
  } | null;
};

async getFollowedVariantsDynamic(userId: string): Promise<FollowedVariantDynamic[]> {
  const supabase = await createClient();
  const { data, error } = await supabase
    .from('user_variant_preference')
    .select(`
      variant:variant_id (
        id, country,
        product:product_id (id, name, code, safety_stock),
        inventory:inventory (
          quantity, last_sync_at, daily_sales, est_days,
          warehouse:warehouse_id (id, name, country, type, lead_time_days)
        )
      )
    `)
    .eq('user_id', userId)
    .eq('preference_type', 'favorited');

  // 查询失败 → 抛出错误，由调用方处理
  if (error) {
    throw new PreferenceError('DB_ERROR', `查询关注列表失败: ${error.message}`);
  }

  if (!data || data.length === 0) return [];

  const results: FollowedVariantDynamic[] = [];
  for (const f of data) {
    const variant = unwrapJoin(f.variant);
    if (!variant) continue;
    const product = unwrapJoin(variant.product);
    if (!product) continue;
    // 显式类型：正式实现使用项目 Database 类型替代
    const inventoryRows: InventoryJoinRowC[] = (variant.inventory ?? []) as unknown as InventoryJoinRowC[];

    for (const inv of inventoryRows) {
      const wh = unwrapJoin(inv.warehouse);
      if (!wh) continue;

      const safetyStock = product.safety_stock ?? 0;
      // 阶段 C 动态告警（主：est_days < lead_time_days，兜底：quantity < safety_stock）
      const isLow = isLowStock(inv.quantity, inv.est_days ?? null, wh.lead_time_days ?? null, safetyStock);

      let alertReason: string | null = null;
      if (inv.est_days != null && wh.lead_time_days != null && inv.est_days < wh.lead_time_days) {
        alertReason = `可售${inv.est_days}天 补货需${wh.lead_time_days}天`;
      } else if (inv.quantity < safetyStock) {
        alertReason = `低于安全线 ${safetyStock}`;
      }

      results.push({
        variantId: variant.id, productName: product.name, productCode: product.code,
        country: variant.country, warehouseId: wh.id, warehouseName: wh.name,
        quantity: inv.quantity, safetyStock,
        dailySales: inv.daily_sales ?? null,
        estDays: inv.est_days ?? null,
        leadTimeDays: wh.lead_time_days ?? null,
        isLowStock: isLow, alertReason,
      });
    }
  }

  results.sort((a, b) => Number(a.isLowStock) < Number(b.isLowStock) ? 1 : -1);
  return results;
}
```

### 6.2 数据库层 — Migration

#### 6.2.1 user_warehouses 表

```sql
CREATE TABLE user_warehouses (
  user_id       uuid        NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  warehouse_id  uuid        NOT NULL REFERENCES warehouse(id) ON DELETE CASCADE,
  created_at    timestamptz NOT NULL DEFAULT now(),
  PRIMARY KEY (user_id, warehouse_id)
);

CREATE INDEX idx_user_warehouses_warehouse_id ON user_warehouses(warehouse_id);

ALTER TABLE user_warehouses ENABLE ROW LEVEL SECURITY;

CREATE POLICY "admin_all_user_warehouses" ON user_warehouses
  FOR ALL
  USING (get_user_role() = 'admin')
  WITH CHECK (get_user_role() = 'admin');

CREATE POLICY "operator_select_own_warehouses" ON user_warehouses
  FOR SELECT
  USING (get_user_role() = 'operator' AND auth.uid() = user_id);

-- 默认全分配现有 operator
INSERT INTO user_warehouses (user_id, warehouse_id)
SELECT p.id, w.id
FROM profiles p
CROSS JOIN warehouse w
JOIN role r ON r.id = p.role_id
WHERE r.name = 'operator'
  AND p.is_active = true
ON CONFLICT (user_id, warehouse_id) DO NOTHING;
```
